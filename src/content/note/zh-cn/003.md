# 准备工作

确保你已经具备以下工具：

- Git  
- VSCode（或任意文本编辑器）  
- Node.js  
- GitHub 账号  
- Netlify 账号  

这些都是基础配置，网上有大量相关教程，不再赘述。

## 配置 GitHub SSH Key

### 检查是否已有 SSH Key

打开终端（以 PowerShell 为例）：

```bash
cd ~/.ssh
ls
# 查看是否存在 id_rsa 和 id_rsa.pub
```

若已存在，可直接跳至[添加 SSH Key](./003#添加-ssh-key)。

### 生成新的 SSH Key

```bash
ssh-keygen -t rsa -C "Your-Email"
# 执行后一路回车即可
```

其中 `"Your-Email"` 为你注册 GitHub 时使用的邮箱。

> [!note]
> 双引号不能省略！  
> 例如：
> ```bash
> ssh-keygen -t rsa -C "1145147454088@qq.com"
> ```

### 添加 SSH Key 到 GitHub

```bash
notepad ./.ssh/id_rsa.pub
```

复制 `id_rsa.pub` 的全部内容。  
打开 [GitHub SSH 设置页面](https://github.com/settings/keys)，点击 `New SSH key`。  
`Title` 随意填写，`Key` 栏粘贴刚复制的内容，点击 `Add SSH key`。

### 验证 SSH Key

执行：

```bash
ssh -T git@github.com
```

若出现提示：

```
Hi [Your GitHub Name]! You've successfully authenticated, but GitHub does not provide shell access.
```

说明配置成功。

# 关于包管理器

本人在使用 npm 时屡屡受挫，因此转向了更高效的 **bun**。  
为了兼容不同用户，文中会同时给出 npm 与 bun 的命令。  
若你尚未安装 bun，可通过 npm 安装：

```bash
npm install -g bun
bun --version
```

若能正确输出版本号，即安装成功。

# 安装 ThoughtLite 主题

npm:
```bash
npm create astro@latest --template tuyuritio/astro-theme-thought-lite

# 根据提示创建项目
cd <your-project-name>
npm run dev
```

bun:
```bash
bun create astro@latest --template tuyuritio/astro-theme-thought-lite

# 根据提示创建项目
cd <your-project-name>
bun run dev
```

# 配置 ThoughtLite

## `.env`

```bash
cp .env.example .env
```

| 变量 | 说明 |
| - | - |
| `PUBLIC_TIMEZONE` | 显示时区，详见 [时区列表](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones#List) |

## `astro.config.ts`

配置项 `site` 为你的站点域名。

## `site.config.ts`

主要参数说明：

- `title`：站点标题  
- `prologue`：首页标语，支持 `\n` 换行  
  > [!note]
  > 例：`"It's just finding words to match words,\none person thinking with another.\n But in this process, it seems we can do something."`
- `author`：作者信息（字符串或对象）  
- `description`：站点描述  
- `copyright`：版权信息与年份  
- `feed`：订阅源配置  
- `latest`：首页最新内容模块配置  

## 图标生成

推荐使用 [RealFaviconGenerator](https://realfavicongenerator.net/)  
生成的文件全部覆盖至 `/public` 目录。  
常见文件包括：

- `apple-touch-icon.png`  
- `favicon.ico`  
- `favicon.svg`  
- `site.webmanifest` 等。

## 首页 Logo

位于 `src/pages/[...locale]/index.astro` 中：

```html
<Icon name="site-logo" size={100} is:inline />
```

你可通过以下方式自定义：

1. 替换 `src/icons/site-logo.svg`  
2. 使用 Iconify 图标集（格式为 `<prefix>:<icon>`）  
3. 改为本地图像或删除该部分。

# 部署网站

## 推送到 GitHub

在 GitHub 创建一个新仓库（建议设为 Public）。

回到项目根目录，执行：

```bash
git remote add origin git@github.com:[YourName]/[RepoName].git
git branch -M main
```

示例：
```bash
git remote add origin git@github.com:Yaten-Z/Try.git
git branch -M main
```

然后：

```bash
bun install netlify
bun run build # 或 npm run build
```

最后推送：

```bash
git add .
git commit -m "init blog"
git push -u origin main
```

## 部署到 Netlify

打开 [Netlify](https://app.netlify.com/)，选择  
`Add new project → Import an existing project → GitHub`  

若未显示仓库，可选择  
`Configure the Netlify app on GitHub` 并授权。  

选择你的仓库 → 填写项目名 → 点击 `Deploy Blog`  
部署完成后，你即可通过  
`https://[Your Project Name].netlify.app`  
访问你的网站。

## 自定义域名（可选）

购买域名后，在 Netlify 项目管理中：  
进入 `Domain management` → 点击 `Add domain alias` → 输入你的域名。  

如：`https://blog.yaten.xyz` → 点击 `Save`

然后在域名管理界面中添加解析：

```
| CNAME | @ | 默认 | [Your Project Name].netlify.app |
```

等待生效后，你的自定义域名即可访问博客。
